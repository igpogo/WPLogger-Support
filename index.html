WP Logger

    Guide for Almond/Walnut/Vine

      Functionality

        The app logs and gives advice on some crops to whether it is time to water or wait to water the fields with the readings collected with a water potential chamber.

      Description

        WP Logger helps you keep track of your crop and guides you with the necessary treatment to achieve its potential yield . Data Logging: The application will record and make it easy to share your collected data. You will be able to locate your measures on the field providing points to collect your data and make decisions. Data Visualization: get your measurements on a map to understand the variability on your fields. Irrigation Assessment: crops that are on the application will assist in irrigation pattern and schedules.

      Use

        The application saves measurements, location, and weather data, temperature and humidity. With these values is possible to calculate the ideal value for the Stem Water Potential measure of a plant measure under those temperature and humidity values. These measures can be used to decide whether or not the plants need to be watered. The user chooses to create a new Session Logging. The application shows a Logging View. In this view the user can choose the crop and temperature and humidity. The user can annotate the value measured with the pressure chamber. On the view there is a button “Next”, which creates a new Logging View and a button “Done” which finishes the Session Logging showing next the Detail View On the Detail View the user can see the averaged values for Stem water Potential, ideal Stem Water potential, temperature, and humidity. There is a “Crop” Button, “Measures” Button and a “Map” Button Pressing on “Crop” Button the user will be able to choose another crop with those measurements “Measures” Button shows the number of measurements on the button’s title as “XXX measurements” pressing on it will show the Measures List View “Map” Button will bring the Map view on the screen Measures List show the list saved for a given Session, It will be TableView sorted by time pressing on them will bring a detail view of the measure as a Logging View but with the fields completed, and no “Done” Button. The view will show the “Next” Button where the user can add more measures to an existing session The user can edit the values after a prompt is shown with a message instructing on the none reversal effects of changes. Map view will show the location of the measurements overlapped a map. The view will also show the session title, and pressing on the overlapped icon the user will be able to see the values measured, ideal, temperature and humidity
